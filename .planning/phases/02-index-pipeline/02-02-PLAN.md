Create `src/core/chunker.ts`:

1. Define and export the `Chunk` interface:
```typescript
export interface Chunk {
  id: string;           // "{relativePath}::{headingPath}" with collision suffix
  headingPath: string;  // "# H1 > ## H2 > ### H3" or "(root)" for no-heading content
  embeddableText: string; // breadcrumb + "\n\n" + body text
  chunkHash: string;    // SHA-256 of embeddableText
}
```

2. Export `chunkMarkdown(content: string, relativePath: string, maxTokens?: number): Chunk[]`

Implementation details:
- Parse with `unified().use(remarkParse).use(remarkFrontmatter, ['yaml']).use(remarkGfm).parse(content)`
- Walk the `Root` node's `children` array sequentially
- Maintain a heading stack: `[h1Text, h2Text, h3Text]`. On encountering a heading node depth 1-3:
  - Flush the current accumulated body nodes as a chunk (if any body exists)
  - Update the heading stack (pop to correct depth, push new heading)
- Heading nodes depth 4+ are treated as body content (accumulated, not split)
- Frontmatter nodes (type === 'yaml') are skipped entirely
- Use `mdast-util-to-string` to extract plain text from body AST nodes
- `maxTokens` defaults to 500. Estimate tokens as `Math.ceil(text.length / 4)`
- For files with no headings: if estimated tokens <= maxTokens, one chunk with headingPath "(root)". If over, split content at double newlines (find `\n\n` boundaries in the raw text extracted from AST) and create numbered chunks with headingPath "(root:1)", "(root:2)", etc.

Chunk ID construction:
- Base: `${relativePath}::${headingPath}`
- Track a Map<string, number> of seen IDs per call. If collision, append `-2`, `-3`, etc.
- Example: `"20 - Journal/2026-04-05.md::# Daily Note > ## Tasks"` 

Hashing:
```typescript
import { createHash } from 'crypto';
function sha256(text: string): string {
  return createHash('sha256').update(text, 'utf-8').digest('hex');
}
```

Breadcrumb format: `"# H1 > ## H2 > ### H3"` -- the `#` markers indicate depth, `>` separates levels.
Embeddable text: `"${breadcrumb}\n\n${bodyText}"` -- breadcrumb is the first line, then blank line, then content.
